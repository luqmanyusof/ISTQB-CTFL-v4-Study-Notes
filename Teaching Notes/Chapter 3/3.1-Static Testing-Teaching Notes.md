# Teaching Notes: 3.1 Static Testing Basics

![Review vs Execution](https://commons.wikimedia.org/wiki/File:Controller_Testing_Steps_Diagram.png)

## Static Testing Methods

| Method | Software Example |
|--------|------------------|
| Manual Review | Team reviews requirements document for gaps |
| Static Analysis | SonarQube scans code for security vulnerabilities |

---

## Work Products for Static Testing

| Work Product | Software Example |
|--------------|------------------|
| Requirements | Review user story for missing acceptance criteria |
| Source Code | Analyze code for unused variables |
| Test Cases | Review test cases for missing edge cases |
| Design Docs | Review API design for inconsistencies |

---

## Value of Static Testing

| Value | Software Example |
|-------|------------------|
| Early Detection | Find missing field validation in requirements |
| Find Hidden Defects | Detect unreachable code branch |
| Cost Savings | Fix design flaw before coding starts |
| Improved Communication | Team aligns on requirements during review |

---

## Static vs Dynamic Testing

| Aspect | Static Example | Dynamic Example |
|--------|----------------|-----------------|
| Execution | Review code without running | Run application and test |
| Defect Finding | See null check missing in code | App crashes on null input |
| Work Products | Review requirements document | Test running application |

---

## Defects Found by Static Testing

| Defect Type | Software Example |
|-------------|------------------|
| Requirements | Ambiguous: "fast response" not defined |
| Design | Database schema missing required index |
| Coding | Variable declared but never used |
| Standards | Function name violates naming convention |
| Interface | API expects string, caller sends integer |
| Security | SQL query built with string concatenation |

---
