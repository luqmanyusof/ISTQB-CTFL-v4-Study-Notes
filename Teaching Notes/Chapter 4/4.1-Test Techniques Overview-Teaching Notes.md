# Teaching Notes: 4.1 Test Techniques Overview

## Test Technique Categories

| Category | Software Example |
|----------|------------------|
| Black-box | Test login based on requirements only |
| White-box | Test all code branches in validation logic |
| Experience-based | Test areas that often have bugs |

---

## Choosing Test Techniques

| Factor | Software Example |
|--------|------------------|
| Component Type | API needs boundary value testing |
| Regulatory | Medical software requires full coverage |
| Risk Level | Payment module needs thorough testing |
| Time Available | Limited time: use risk-based approach |
| Tester Skills | Team knows exploratory testing well |

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Black-box Technique | Test based on specification, ignore code |
| White-box Technique | Test based on code structure |
| Experience-based | Test where bugs were found before |
| Technique Selection | High-risk feature needs multiple techniques |
