# Teaching Notes: 4.4 Experience-based Test Techniques

## Error Guessing

- **Concept:** Predict bugs based on tester's experience
- **Example:** Test null input—developers often forget null checks

| Common Error | Software Example |
|--------------|------------------|
| Null/Empty | Submit form with empty required fields |
| Zero Division | Calculate average with zero items |
| Boundary | Enter 0 or negative quantity |
| Special Chars | Input <script> tag in text field |

---

## Exploratory Testing

- **Concept:** Simultaneous learning, test design, and execution
- **Example:** Explore new feature without predefined test cases

| Element | Software Example |
|---------|------------------|
| Charter | "Explore checkout with various payment methods" |
| Time-box | 60-minute session testing search functionality |
| Notes | Document unexpected behavior found during session |

---

## Checklist-based Testing

- **Concept:** Use checklist as guide, not detailed test cases
- **Example:** Security checklist: SQL injection, XSS, CSRF

| Checklist Item | Software Example |
|----------------|------------------|
| Input Validation | Check all form fields reject invalid input |
| Error Handling | Verify user-friendly error messages shown |
| Performance | Confirm page loads under 3 seconds |

---
