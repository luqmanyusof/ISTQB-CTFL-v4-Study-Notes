# Teaching Notes: 4.3 White-box Test Techniques

## Statement Testing

- **Concept:** Execute every statement in code at least once
- **Example:** Ensure every line in calculateDiscount() runs

| Coverage | Software Example |
|----------|------------------|
| 100% Statement | All code lines executed during tests |
| Gap Found | Line inside rare if-condition never executed |

---

## Branch Testing

- **Concept:** Execute every branch (true/false) at least once
- **Example:** Test both if(isAdmin) true and false paths

| Coverage | Software Example |
|----------|------------------|
| Branch True | Test when user.isAdmin = true |
| Branch False | Test when user.isAdmin = false |
| 100% Branch | All if/else paths executed |

---

## Statement vs Branch Coverage

| Aspect | Software Example |
|--------|------------------|
| Statement Only | Execute if-block, miss else-block |
| Branch Complete | Execute both if-block and else-block |
| Relationship | 100% branch implies 100% statement |

---

## Quick Reference (≤10 words each)

| Technique | Software Example |
|-----------|------------------|
| Statement Testing | Execute every line in validation function |
| Statement Coverage | 85% of code lines executed by tests |
| Branch Testing | Test both true and false conditions |
| Branch Coverage | All if/else paths tested at least once |
| Coverage Gap | Error handling code never executed in tests |
