# Teaching Notes: 4.5 Collaboration-based Test Approaches

## Collaborative User Story Writing

- **Concept:** Team writes user stories together
- **Example:** Dev, tester, PO define "As user, I can reset password"

---

## Acceptance Criteria

- **Concept:** Conditions that must be met for story completion
- **Example:** "Password reset email sent within 30 seconds"

| Format | Software Example |
|--------|------------------|
| Given/When/Then | Given logged out, When click reset, Then email sent |
| Checklist | Email contains reset link, link expires in 24h |

---

## ATDD (Acceptance Test-Driven Development)

- **Concept:** Write acceptance tests before development
- **Example:** Define "user sees error for invalid email" before coding

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| User Story | "As user, I can filter products by price" |
| Acceptance Criteria | Filter shows only products in selected range |
| Given/When/Then | Given products exist, When filter applied, Then filtered |
| ATDD | Write acceptance test before implementing feature |
| Three Amigos | Dev, tester, PO discuss story requirements together |
