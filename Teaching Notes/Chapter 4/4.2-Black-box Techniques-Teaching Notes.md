# Teaching Notes: 4.2 Black-box Test Techniques

## Equivalence Partitioning (EP)

- **Concept:** Divide inputs into groups that behave the same
- **Example:** Age field: valid (18-65), invalid (<18), invalid (>65)

| Partition | Software Example |
|-----------|------------------|
| Valid | Email with @ and domain: user@test.com |
| Invalid | Email without @: usertest.com |

---

## Boundary Value Analysis (BVA)

- **Concept:** Test at edges of equivalence partitions
- **Example:** Age 17, 18, 65, 66 for range 18-65

| Boundary | Software Example |
|----------|------------------|
| Min-1 | Password length 7 (min is 8) |
| Min | Password length 8 |
| Max | Password length 20 |
| Max+1 | Password length 21 |

---

## Decision Table Testing

- **Concept:** Test combinations of conditions and actions
- **Example:** Discount rules: member + coupon + order>$100

| Conditions | Software Example |
|------------|------------------|
| Rule 1 | Premium member + valid coupon = 20% off |
| Rule 2 | Regular member + valid coupon = 10% off |
| Rule 3 | Any member + expired coupon = no discount |

---

## State Transition Testing

- **Concept:** Test system behavior across different states
- **Example:** Order states: pending → paid → shipped → delivered

| Transition | Software Example |
|------------|------------------|
| Valid | Order: pending → paid (payment received) |
| Invalid | Order: pending → delivered (skip payment) |
| State | Login: 3 failed attempts → account locked |

---

## Quick Reference (≤10 words each)

| Technique | Software Example |
|-----------|------------------|
| Equivalence Partitioning | Group: valid emails, invalid emails, empty field |
| Boundary Value | Test quantity: 0, 1, 99, 100, 101 |
| Decision Table | Combine: user type + subscription + region |
| State Transition | Test: draft → published → archived flow |
