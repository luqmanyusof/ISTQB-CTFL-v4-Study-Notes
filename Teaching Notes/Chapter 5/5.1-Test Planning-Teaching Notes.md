# Teaching Notes: 5.1 Test Planning

## Test Plan Purpose

- **Concept:** Document describing scope, approach, resources, schedule
- **Example:** Plan defines testing payment module in Sprint 5

---

## Test Plan Content

| Element | Software Example |
|---------|------------------|
| Scope | Test checkout, payment, order confirmation |
| Approach | Automated API tests + manual UI tests |
| Resources | 2 testers, Selenium, Postman |
| Schedule | Week 1: test cases, Week 2: execution |
| Entry Criteria | Code deployed to test environment |
| Exit Criteria | 95% test cases passed, no critical bugs |

---

## Risk-based Testing

- **Concept:** Prioritize testing based on risk level
- **Example:** Payment module high risk, about page low risk

| Risk Level | Software Example |
|------------|------------------|
| High | Payment processing—test extensively |
| Medium | User profile—moderate testing |
| Low | Static FAQ page—basic testing |

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Test Plan | Document defining what, how, when to test |
| Scope | Features included: login, checkout, reports |
