# Teaching Notes: 5.4 Configuration Management

## Configuration Management Purpose

- **Concept:** Control and track versions of test items
- **Example:** Track which test cases apply to version 2.1

---

## Configuration Items

| Item | Software Example |
|------|------------------|
| Test Plans | Test plan v1.0, v1.1, v2.0 |
| Test Cases | Test case TC-101 revision 3 |
| Test Scripts | Selenium script for login v2 |
| Test Data | Test database snapshot 2024-01-15 |
| Test Results | Test run results for build 456 |

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Version Control | Test scripts stored in Git repository |
| Baseline | Approved test suite for release 3.0 |
| Traceability | Link test case to requirement and defect |
| Change Control | Approve test case changes before execution |
