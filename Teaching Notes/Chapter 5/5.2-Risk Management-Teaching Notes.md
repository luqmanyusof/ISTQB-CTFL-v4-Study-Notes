# Teaching Notes: 5.2 Risk Management

## Risk Types

| Type | Software Example |
|------|------------------|
| Product Risk | Payment calculation bug causes revenue loss |
| Project Risk | Key tester leaves mid-project |

---

## Risk Assessment

| Factor | Software Example |
|--------|------------------|
| Likelihood | High: complex legacy code often has bugs |
| Impact | Critical: payment failure loses customers |
| Risk Level | Likelihood × Impact = Priority |

---

## Risk Mitigation

| Strategy | Software Example |
|----------|------------------|
| Testing | More test cases for high-risk payment module |
| Contingency | Backup tester available if primary unavailable |

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Product Risk | Bug in checkout causes order failures |
| Project Risk | Test environment unavailable delays testing |
| Risk Likelihood | Legacy code: high chance of defects |
| Risk Impact | Security breach: severe business damage |
| Risk Mitigation | Add security testing for authentication module |
