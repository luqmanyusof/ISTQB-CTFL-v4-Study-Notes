# Teaching Notes: 5.3 Test Monitoring and Control

## Test Monitoring

- **Concept:** Track test progress against plan
- **Example:** Dashboard shows 60% tests executed, 5 days left

| Metric | Software Example |
|--------|------------------|
| Test Progress | 150 of 200 test cases executed |
| Defect Status | 12 open bugs, 8 critical |
| Coverage | 85% requirements covered by tests |

---

## Test Control

- **Concept:** Take action when progress deviates from plan
- **Example:** Add tester when behind schedule

| Action | Software Example |
|--------|------------------|
| Re-prioritize | Focus on critical path tests first |
| Add Resources | Assign additional tester to sprint |
| Adjust Scope | Defer low-priority feature testing |

---

## Test Reports

| Report Type | Software Example |
|-------------|------------------|
| Progress Report | Daily: tests run, passed, failed, blocked |
| Summary Report | Final: overall results, recommendations |
