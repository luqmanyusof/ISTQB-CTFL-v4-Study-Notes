# Teaching Notes: 5.5 Defect Management

## Defect Report Content

| Field | Software Example |
|-------|------------------|
| ID | BUG-1234 |
| Title | Login fails with valid credentials |
| Steps to Reproduce | 1. Enter valid email 2. Enter valid password 3. Click login |
| Expected Result | User redirected to dashboard |
| Actual Result | Error message "Invalid credentials" shown |
| Severity | Critical |
| Priority | High |
| Environment | Chrome 120, Windows 11, Production |

---

## Defect Lifecycle

| Status | Software Example |
|--------|------------------|
| New | Tester reports bug in Jira |
| Open | Developer assigned to fix |
| In Progress | Developer working on fix |
| Fixed | Developer commits fix to branch |
| Ready for Test | Fix deployed to test environment |
| Verified | Tester confirms bug is fixed |
| Closed | Bug resolved and closed |
| Reopened | Bug still occurs after fix |

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Defect Report | Document bug with steps to reproduce |
| Severity | Critical: system crash; Minor: typo |
| Priority | High: fix before release; Low: fix later |
| Defect Status | New → Open → Fixed → Verified → Closed |
| Root Cause | Missing null check caused crash |
