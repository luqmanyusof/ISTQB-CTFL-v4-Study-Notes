# Teaching Notes: 2.1 Testing in the Context of SDLC

## SDLC Models with Software Examples

| SDLC Type | Software Example |
|-----------|------------------|
| Sequential (Waterfall) | Complete all requirements before any coding starts |
| Iterative | Build login module, refine it, then build dashboard |
| Incremental | Release v1 with basic features, v2 adds more |
| Agile (Scrum) | 2-week sprints delivering working features each time |

---

## Good Testing Practices (All SDLCs)

| Practice | Software Example |
|----------|------------------|
| Correspondence | Design phase → design review; coding → unit tests |
| Unique Objectives | Unit tests find code bugs; UAT validates business needs |
| Early Analysis | Write test cases while requirements are drafted |
| Early Review | Review user stories before sprint starts |

---

## Test-First Approaches

| Approach | Software Example |
|----------|------------------|
| TDD | Write failing unit test, then write code to pass |
| ATDD | Define "user can reset password" test before coding |
| BDD | Given logged in, When click logout, Then redirected |

---

## DevOps and Testing

| Aspect | Software Example |
|--------|------------------|
| CI Benefit | Automated tests run on every git push |
| CD Benefit | Deploy to staging automatically after tests pass |
| Fast Feedback | Developer sees test failure within 5 minutes |
| Challenge | Maintaining 500+ automated tests in pipeline |

---

## Shift Left

- **Concept:** Test earlier in SDLC, not just at the end
- **Example:** Review requirements for testability before coding

| Practice | Software Example |
|----------|------------------|
| Review Specs | Find missing validation rules in requirements |
| Write Tests First | Create API test cases before implementation |
| Static Analysis | Run SonarQube before code review |
| Early Non-functional | Load test component before integration |

---

## Retrospectives

- **Concept:** Team reflects on what worked and what to improve
- **Example:** "We missed bugs because test data was incomplete"

| Discussion Point | Software Example |
|------------------|------------------|
| What worked | Automated regression caught 3 bugs early |
| What to improve | Need more edge case test data |
| Action item | Add negative test cases to test suite |

---
