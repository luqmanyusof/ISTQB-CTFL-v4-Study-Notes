# Teaching Notes: 2.3 Maintenance Testing

## Maintenance Categories with Software Examples

| Category | Software Example |
|----------|------------------|
| Corrective | Fix production bug causing login failures |
| Adaptive | Update app to support new iOS version |
| Perfective | Optimize database queries for faster loading |

---

## Triggers for Maintenance Testing

| Trigger | Software Example |
|---------|------------------|
| Modifications | Add new payment method to checkout |
| Upgrades/Migrations | Migrate database from MySQL to PostgreSQL |
| Retirement | Archive old CRM data before system shutdown |

---

## Impact Analysis

- **Concept:** Identify what could be affected by changes
- **Example:** Changing user table affects login, profile, orders

---

## Quick Reference (≤10 words each)

| Concept | Software Example |
|---------|------------------|
| Corrective Maintenance | Hotfix deployed for critical security vulnerability |
| Adaptive Maintenance | Update API for new third-party service version |
| Perfective Maintenance | Refactor code to improve page load speed |
| Modification Trigger | New feature added to existing production system |
| Migration Trigger | Move from on-premise to cloud hosting |
| Retirement Trigger | Export user data before legacy system shutdown |
| Impact Analysis | Check which modules use modified authentication |
| Regression Scope | Test all features using changed database schema |
