## Test Levels with Software Examples

### Component Testing (Unit Testing)
- **Concept:** Test individual units in isolation
- **Example:** Test calculateTotal() function returns correct sum

### Component Integration Testing
- **Concept:** Test interfaces between components
- **Example:** State dropdown updates when country dropdown changes

### System Testing
- **Concept:** Test complete system end-to-end
- **Example:** Complete checkout flow from cart to confirmation

### System Integration Testing
- **Concept:** Test system with external systems/services
- **Example:** App correctly sends data to payment gateway API

### Acceptance Testing
- **Concept:** Validate system meets business needs
- **Example:** Business user confirms report shows correct data

## Acceptance Testing Forms

| Form | Software Example |
|------|------------------|
| UAT | Business user tests invoice generation workflow |
| Operational | Ops team verifies backup and restore works |
| Contractual | Client verifies all contracted features exist |
| Regulatory | Verify GDPR data deletion works correctly |
| Alpha | Internal staff tests new mobile app version |
| Beta | Selected customers test app before public release |

---

## Test Types with Software Examples

### Functional Testing
- **Concept:** Test WHAT the system does
- **Example:** Login accepts valid credentials, rejects invalid

### Non-functional Testing
- **Concept:** Test HOW WELL the system performs
- **Example:** Page loads in under 2 seconds

### Black-box Testing
- **Concept:** Test based on specifications, not code
- **Example:** Test login without knowing authentication code

### White-box Testing
- **Concept:** Test based on internal code structure
- **Example:** Test all branches in if-else statement

---

## Confirmation vs Regression Testing

| Type | Software Example |
|------|------------------|
| Confirmation | Re-run failed login test after bug fix |
| Regression | Run full test suite after any code change |
