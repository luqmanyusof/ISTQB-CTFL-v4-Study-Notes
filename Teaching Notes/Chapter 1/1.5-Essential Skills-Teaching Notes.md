# Teaching Notes: 1.5 Essential Skills and Good Practices

## Generic Skills with Software Examples

| Skill | Software Example |
|-------|------------------|
| Testing Knowledge | Apply boundary value analysis to age field |
| Attention to Detail | Notice button misaligned by 2 pixels |
| Communication | Write clear bug report with steps to reproduce |
| Analytical Thinking | Identify edge case: what if cart is empty? |
| Technical Knowledge | Use browser DevTools to inspect API calls |
| Domain Knowledge | Understand banking rules for transaction limits |

---

## Whole Team Approach

- **Concept:** Everyone responsible for quality, not just testers
- **Example:** Developer writes unit tests before code review

---

## Independence of Testing

| Level | Software Example |
|-------|------------------|
| No Independence | Developer tests own code |
| Some Independence | Peer developer reviews and tests code |
| High Independence | Dedicated QA team tests the feature |
| Very High Independence | External security audit firm tests app |

### Benefits of Independence
- **Example:** External tester finds bug developer overlooked

### Drawbacks of Independence
- **Example:** Separate QA team causes "throw over wall" mentality
