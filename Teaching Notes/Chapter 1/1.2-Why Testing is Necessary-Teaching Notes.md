# Teaching Notes: 1.2 Why Testing is Necessary?

## Key Concepts with Software Examples

### Testing's Contribution to Success
- **Concept:** Testing reduces risk and supports release decisions
- **Example:** Load testing prevents Black Friday website crash

### Testing vs Quality Assurance (QA)
- **Concept:** Testing = product-focused (corrective); QA = process-focused (preventive)
- **Example (Testing):** Finding bugs in deployed feature
- **Example (QA):** Enforcing code review before merge

### Error → Defect → Failure Chain
- **Concept:** Human mistake creates bug, bug causes system failure
- **Example (Error):** Developer types "==" instead of "==="
- **Example (Defect):** Wrong comparison operator in code
- **Example (Failure):** User sees incorrect calculation result

### Root Cause Analysis
- **Concept:** Finding fundamental reason to prevent recurrence
- **Example:** Crash caused by missing null check—add validation
