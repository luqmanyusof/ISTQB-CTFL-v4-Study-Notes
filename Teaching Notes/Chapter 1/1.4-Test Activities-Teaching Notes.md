# Teaching Notes: 1.4 Test Activities, Testware and Test Roles

## Test Activities with Software Examples

### 1. Test Planning
- **Concept:** Define scope, approach, resources, schedule
- **Example:** Plan to test checkout flow with 3 testers, 2 weeks

### 2. Test Monitoring and Control
- **Concept:** Track progress; take action when off-track
- **Example (Monitoring):** 60% test cases executed, 2 days left
- **Example (Control):** Add testers to meet deadline

### 3. Test Analysis (What to test)
- **Concept:** Identify testable features from requirements
- **Example:** Requirement says "user can filter by date"—test filter

### 4. Test Design (How to test)
- **Concept:** Create test cases from test conditions
- **Example:** Design: input valid date, invalid date, empty date

### 5. Test Implementation
- **Concept:** Prepare everything needed for execution
- **Example:** Set up test database with sample user accounts

### 6. Test Execution
- **Concept:** Run tests and compare actual vs expected
- **Example:** Run login test; expected "Welcome", got "Error 500"

### 7. Test Completion
- **Concept:** Archive testware, create reports, lessons learned
- **Example:** Document that API tests need more edge cases

## Testware Examples

| Activity | Testware Example |
|----------|------------------|
| Planning | Test plan document, risk register |
| Analysis | List of test conditions for search feature |
| Design | Test cases for password validation |
| Implementation | Automated Selenium scripts, test data files |
| Execution | Test logs, defect reports in Jira |
| Completion | Test summary report, archived test suites |

## Roles in Testing

| Role | Software Example |
|------|------------------|
| Test Manager | Plans sprint testing, assigns testers to features |
| Tester | Writes and executes test cases for login |

---

## Traceability

- **Concept:** Link requirements → test cases → defects
- **Example:** REQ-101 (login) → TC-201, TC-202 → BUG-301
