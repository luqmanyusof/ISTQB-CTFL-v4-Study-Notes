# Teaching Notes: 1.1 What is Testing?

## Key Concepts with Software Examples

### Testing Definition
- **Concept:** Testing discovers defects and evaluates quality
- **Example:** Running login tests to find authentication bugs

### Test Object
- **Concept:** The item being tested
- **Example:** API endpoint, web form, database query, mobile screen

### Verification vs Validation
- **Concept:** Verification = meets requirements; Validation = meets user needs
- **Example (Verification):** Login accepts valid email format per spec
- **Example (Validation):** Users actually want social login, not email

### Static vs Dynamic Testing
- **Concept:** Static = no execution; Dynamic = execution required
- **Example (Static):** Code review finds null pointer before running
- **Example (Dynamic):** Running app crashes on null input

### Testing vs Debugging
- **Concept:** Testing finds failures; Debugging fixes root cause
- **Example (Testing):** Tester reports "checkout button unresponsive"
- **Example (Debugging):** Developer finds missing event listener, fixes it

### Confirmation vs Regression Testing
- **Concept:** Confirmation = fix works; Regression = no side effects
- **Example (Confirmation):** Re-test checkout button after fix
- **Example (Regression):** Verify cart and payment still work after fix
