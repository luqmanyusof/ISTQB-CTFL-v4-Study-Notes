## 🔎 What are Test Techniques?

Test techniques support the tester in:
- **Test analysis:** Determining what to test
- **Test design:** Determining how to test

**Purpose of Test Techniques:**
- Develop a relatively small, but sufficient, set of test cases in a systematic way
- Help define test conditions, identify coverage items, and identify test data

> Further information on test techniques can be found in the ISO/IEC/IEEE 29119-4 standard.

## 📊 Classification of Test Techniques `[K2]`

*FL-4.1.1: Distinguish black-box test techniques, white-box test techniques and experience-based test techniques*

| Technique Type | Also Known As | Based On | Test Case Dependency |
|----------------|---------------|----------|---------------------|
| **Black-box** | Specification-based | Analysis of specified behavior without reference to internal structure | Independent of implementation; useful if behavior stays same |
| **White-box** | Structure-based | Analysis of test object's internal structure and processing | Dependent on design; can only be created after design/implementation |
| **Experience-based** | — | Knowledge and experience of testers | Depends heavily on tester's skills |

---

## ⬛ Black-box Test Techniques

**Definition:** Based on analysis of the specified behavior of the test object **without reference to its internal structure**.

**Key Characteristics:**
- Test cases are **independent of how the software is implemented**
- If implementation changes but required behavior stays the same, **test cases are still useful**
- Focus on **what** the system should do

**Techniques covered in this syllabus:**
- Equivalence Partitioning (EP)
- Boundary Value Analysis (BVA)
- Decision Table Testing
- State Transition Testing

---

## ⬜ White-box Test Techniques

**Definition:** Based on analysis of the test object's **internal structure and processing**.

**Key Characteristics:**
- Test cases are **dependent on how the software is designed**
- Can only be created **after the design or implementation** of the test object
- Focus on **how** the system works internally

**Techniques covered in this syllabus:**
- Statement Testing
- Branch Testing

> More rigorous white-box techniques exist for safety-critical, mission-critical, or high-integrity environments.

---

## 🧠 Experience-based Test Techniques

**Definition:** Effectively use the **knowledge and experience of testers** for design and implementation of test cases.

**Key Characteristics:**
- Effectiveness depends heavily on the **tester's skills**
- Can detect defects that may be **missed by black-box and white-box techniques**
- **Complementary** to black-box and white-box techniques

**Techniques covered in this syllabus:**
- Error Guessing
- Exploratory Testing
- Checklist-based Testing

---

## 🎯 Exam Prep Tips

- **Three Categories `[K2]`:** Know the three categories: black-box (specification-based), white-box (structure-based), experience-based.
- **Black-box `[K2]`:** Independent of implementation; based on specified behavior; test cases remain valid if implementation changes.
- **White-box `[K2]`:** Dependent on implementation; can only be created after design/implementation.
- **Experience-based `[K2]`:** Depends on tester's skills; complementary to other techniques.
- **Complementary:** All three types complement each other for comprehensive testing.
