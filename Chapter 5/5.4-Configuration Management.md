## 🔎 What is Configuration Management?

**Configuration Management (CM)** in testing provides a discipline for identifying, controlling, and tracking work products as **configuration items**.

### Configuration Items in Testing
- Test plans
- Test strategies
- Test conditions
- Test cases
- Test scripts
- Test results
- Test logs
- Test reports

## 📦 5.4.1 How Configuration Management Supports Testing `[K2]`

*FL-5.4.1: Summarize how configuration management supports testing*

### Key Concepts

| Concept | Description |
|---------|-------------|
| **Configuration Item** | Any work product that is tracked and version controlled |
| **Baseline** | Approved configuration item that can only be changed through formal change control process |
| **Complex Configuration Item** | CM records the items it consists of, their relationships, and versions (e.g., test environment) |

### What CM Ensures for Testing

| Capability | Description |
|------------|-------------|
| **Unique Identification** | All configuration items, including test items (individual parts of test object), are uniquely identified |
| **Version Control** | All items are version controlled |
| **Change Tracking** | Changes are tracked |
| **Traceability** | Items are related to other configuration items so traceability can be maintained throughout the test process |
| **Unambiguous References** | All identified documentation and software items are referenced unambiguously in testware |

### Reverting to Previous Baseline
- CM keeps a record of changed configuration items when a new baseline is created
- It is possible to **revert to a previous baseline** to reproduce previous test results

### CM in DevOps
- Continuous Integration (CI), Continuous Delivery (CD), Continuous Deployment and associated testing are typically implemented as part of an **automated DevOps pipeline**
- **Automated CM** is normally included in DevOps pipelines

- **Traceability `[K2]`:** CM maintains traceability between configuration items throughout the test process.
- **Reproducibility `[K2]`:** CM allows reverting to previous baselines to reproduce previous test results.
