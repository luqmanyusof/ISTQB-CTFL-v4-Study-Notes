|----------|---------|

| **Test Control** | Using monitoring information to provide guidance and corrective actions for effective testing |
| **Test Completion** | Collecting data from completed test activities to consolidate experience, testware, and relevant information |

## 📊 5.3.1 Metrics Used in Testing `[K1]`

*FL-5.3.1: Recall metrics used for testing*

### Purpose of Test Metrics
- Show progress against planned test schedule and budget
- Show current quality of the test object
- Show effectiveness of test activities with respect to test objectives

### Common Test Metrics

| Category | Examples |
|----------|----------|
| **Project Progress Metrics** | Task completion, resource usage, test effort |
| **Test Progress Metrics** | Test case implementation progress, test environment preparation progress, number of test cases run/not run, passed/failed, test execution time |
| **Product Quality Metrics** | Availability, response time, mean time to failure |
| **Defect Metrics** | Number and priorities of defects found/fixed, defect density, defect detection percentage |
| **Risk Metrics** | Residual risk level |
| **Coverage Metrics** | Requirements coverage, code coverage |
| **Cost Metrics** | Cost of testing, organizational cost of quality |

## 📝 5.3.2 Purpose, Content and Audience for Test Reports `[K2]`

*FL-5.3.2: Summarize the purposes, content, and audiences for test reports*

### Two Types of Test Reports

| Report Type | Purpose | When Generated |
|-------------|---------|----------------|
| **Test Progress Report** | Support ongoing test control; provide information for modifications to schedule, resources, or test plan | Regularly (daily, weekly, etc.) |
| **Test Completion Report** | Summarize specific test activity (test level, test cycle, iteration); provide information for subsequent testing | At project milestones |

### Test Progress Report Content
- Testing period
- Test progress (ahead or behind schedule), including notable deviations
- Impediments for testing and their workarounds
- Test metrics
- New and changed risks within testing period
- Testing planned for the next period

### Test Completion Report Content
- Test summary
- Testing and product quality evaluation based on original test plan (test objectives and exit criteria)
- Deviations from the test plan (differences from planned schedule, duration, effort)
- Testing impediments and workarounds
- Test metrics based on test progress reports
- Unmitigated risks, defects not fixed
- Lessons learned relevant to testing

### Audience Considerations
- Different audiences require **different information**
- Audience influences **degree of formality** and **frequency** of reporting
- **Within team:** Frequent and informal
- **Test completion:** Set template, occurs only once

> ISO/IEC/IEEE 29119-3 includes templates and examples for test reports.

## 📢 5.3.3 Communicating the Status of Testing `[K2]`

*FL-5.3.3: Exemplify how to communicate the status of testing*

### Communication Options

| Method | Description |
|--------|-------------|
| **Verbal Communication** | Direct communication with team members and stakeholders |
| **Dashboards** | CI/CD dashboards, task boards, burn-down charts |
| **Electronic Channels** | Email, chat |
| **Online Documentation** | Shared documents, wikis |
| **Formal Test Reports** | Test progress reports, test completion reports |

### Considerations
- One or more options can be used
- **More formal communication** may be appropriate for distributed teams (geographical distance, time differences)
- Different stakeholders are interested in **different types of information**
- Communication should be **tailored accordingly**

## 🔄 Test Control Examples

Examples of control directives:
- **Reprioritizing tests** when an identified risk becomes an issue
- **Re-evaluating** whether a test item meets entry/exit criteria due to rework
- **Adjusting test schedule** to address delay in test environment delivery
- **Adding new resources** when and where needed

---

## ⏹️ Test Completion Activities

Test completion occurs at project milestones:
- Test level completed
- Agile iteration finished
- Test project completed (or cancelled)
- Software system released
- Maintenance release completed

## 🎯 Exam Prep Tips

- **Metrics `[K1]`:** Recall the seven categories of metrics (project progress, test progress, product quality, defect, risk, coverage, cost).
- **Test Progress Report `[K2]`:** Know the content (testing period, progress, impediments, metrics, risks, next period plans).
- **Test Completion Report `[K2]`:** Know the content (summary, evaluation, deviations, impediments, metrics, unmitigated risks, lessons learned).
- **Communication Methods `[K2]`:** Know the options (verbal, dashboards, electronic, documentation, formal reports).
- **Audience `[K2]`:** Different audiences need different information; tailor communication accordingly.
