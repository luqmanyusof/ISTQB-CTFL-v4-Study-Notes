## 🔎 What is Maintenance Testing?

Maintenance testing is testing performed on an existing operational system after changes have been made.

**Maintenance categories (per ISO/IEC 14764):**
- **Corrective:** Fixing defects found in production
- **Adaptive:** Adapting to changes in the environment (e.g., new OS, database, hardware)
- **Perfective:** Improving performance or maintainability

---

## 📏 Scope of Maintenance Testing

The scope of maintenance testing typically depends on:

| Factor | Impact |
|--------|--------|
| **Degree of risk** | Higher-risk changes require more extensive testing to ensure system stability and user safety. |
| **Size of existing system** | Larger systems have more potential impact areas requiring broader regression test coverage. |
| **Size of change** | Bigger modifications affect more components and require proportionally more comprehensive testing effort. |

---

## 🔔 Triggers for Maintenance Testing `[K2]`

*FL-2.3.1: Summarize maintenance testing and its triggers*

| Trigger | Description | Testing Focus |
|---------|-------------|---------------|
| **Modifications** | Planned enhancements (release-based), corrective bug fixes, and urgent hot fixes | Validate the change works correctly; regression testing for side effects |
| **Upgrades/Migrations** | Platform changes (new OS, database, hardware) | Test new environment compatibility, changed software behavior, data conversion accuracy |
| **Retirement** | End of application life | Test data archiving integrity, restore/retrieval procedure verification |

### Modifications
- Planned enhancements delivered as part of scheduled releases
- Corrective changes to fix reported defects
- Hot fixes for urgent production issues

### Upgrades/Migrations
- Moving to a new platform or environment
- Testing the new environment itself
- Testing the changed software behavior
- Testing data conversion when migrating data from another application

### Retirement
- When a system reaches end of life
- Testing data archiving if long retention periods are required
- Testing restore and retrieval procedures for archived data

---

## 📊 Impact Analysis

> Impact analysis should be done **before changes** to decide if the change should be made based on potential consequences.

**Purpose of Impact Analysis:**
- Recognize the extent of regression testing needed
- Identify which parts of the system could be affected by changes
- Determine the scope of testing required

**Testing Changes to an Operational System Includes:**
- Evaluating the **success of the implementation** of the change
- Checking for possible **regressions** in parts of the system that remain unchanged (usually most of the system)

- **Regression Testing `[K2]`:** Maintenance testing heavily involves regression testing to check for side effects.
