## Chapter 2.2: Test Levels and Test Types

**Learning Objectives:**
- FL-2.2.1 `[K2]`: Distinguish the different test levels
- FL-2.2.2 `[K2]`: Distinguish the different test types
- FL-2.2.3 `[K2]`: Distinguish confirmation testing from regression testing

---

## 🔎 Definitions

**Test levels** = groups of test activities organized and managed together, performed at a given development phase.

**Test types** = groups of test activities related to specific quality characteristics (can be performed at every test level).

---

## 📊 2.2.1 Test Levels `[K2]`

*FL-2.2.1: Distinguish the different test levels*

| Test Level | Focus | Performed By |
|------------|-------|--------------|
| **Component Testing** (Unit) | Testing components in isolation; requires test harnesses/frameworks | Developers |
| **Component Integration Testing** | Interfaces and interactions between components; depends on integration strategy (bottom-up, top-down, big-bang) | Developers |
| **System Testing** | Overall behavior/capabilities of entire system; functional end-to-end tasks + non-functional testing | Independent test team |
| **System Integration Testing** | Interfaces between system under test and other systems/external services | Test team |
| **Acceptance Testing** | Validation and deployment readiness; system fulfills user's business needs | Intended users |

### Acceptance Testing Forms
- **User Acceptance Testing (UAT):** Business users verify the system meets their needs.
- **Operational Acceptance Testing:** Operations team verifies system can be operated and maintained.
- **Contractual Acceptance Testing:** Verifies system meets contractual requirements.
- **Regulatory Acceptance Testing:** Verifies compliance with laws, regulations, and standards.
- **Alpha Testing:** Testing by internal users at the developer's site.
- **Beta Testing:** Testing by external users at their own sites.

### Distinguishing Attributes
Test levels are distinguished by:
- Test object
- Test objectives
- Test basis
- Defects and failures typically found
- Approach and responsibilities

## 🔍 2.2.2 Test Types `[K2]`

*FL-2.2.2: Distinguish the different test types*

| Test Type | Description |
|-----------|-------------|
| **Functional Testing** | Evaluates *what* the system should do; checks functional completeness, correctness, appropriateness |
| **Non-functional Testing** | Evaluates *how well* the system behaves; checks quality characteristics per ISO/IEC 25010 |
| **Black-box Testing** | Specification-based; derives tests from documentation (not internal structure) |
| **White-box Testing** | Structure-based; derives tests from implementation (code, architecture, workflows) |

### ISO/IEC 25010 Non-functional Characteristics
- Performance efficiency
- Compatibility
- Usability
- Reliability
- Security
- Maintainability
- Portability
- Safety

> All four test types can be applied at all test levels with different focus.

## 🔁 2.2.3 Confirmation Testing and Regression Testing `[K2]`

*FL-2.2.3: Distinguish confirmation testing from regression testing*

| Type | Purpose |
|------|---------|
| **Confirmation Testing** | Confirms original defect is successfully fixed; re-execute failed tests + add new tests for fix changes |
| **Regression Testing** | Confirms no adverse consequences from changes; may affect same component, other components, or connected systems |

### Key Points
- **Impact analysis:** Analyze which parts of the system could be affected by changes to determine regression testing scope.
- **Automation candidate:** Regression test suites run repeatedly across releases, making them ideal candidates for test automation.
- **Early automation:** Start building automated regression tests early in the project and integrate them into Continuous Integration (CI) pipelines.
- **All test levels:** Both confirmation and regression testing are needed at every test level when defects are fixed.

## 🎯 Exam Prep Tips

- **Test Levels `[K2]`:** Memorize all five test levels (component, component integration, system, system integration, acceptance) and their focus.
- **Who Performs What:** Know that developers do component/component integration; test team does system/system integration; users do acceptance.
- **Acceptance Testing Forms `[K2]`:** Know UAT, Operational, Contractual, Regulatory, Alpha, Beta.
- **Test Types `[K2]`:** Understand functional (what it does) vs. non-functional (how well it performs).
- **Black-box vs. White-box `[K2]`:** Black-box = specification-based; White-box = structure-based.
- **Confirmation vs. Regression `[K2]`:** Confirmation = fix verified; Regression = no side effects.
- **Regression Automation:** Know that regression testing is a strong candidate for automation.
