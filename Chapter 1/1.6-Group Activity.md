# 📘 ISTQB CTFL v4.0 – Group Activity
## Chapter 1: Fundamentals of Testing

Will be devided to 3 groups, they will chose 3 objectives from the list below randomly and come up with emojis that explains each objective. They will present the emojis let other groups guess what each emoji represents.

### Emoji Shortcut
 - From 9 objectives in testing please come out with emojis that explains each objective.

The typical test objectives are:
- **Evaluating work products:** Assess requirements, user stories, designs, and code for quality and completeness.
- **Causing failures and finding defects:** Trigger failures through execution to identify underlying defects in the software.
- **Ensuring required coverage:** Verify that the test object is adequately covered by test cases.
- **Reducing risk level:** Lower the probability of inadequate software quality reaching production environments.
- **Verifying requirements:** Confirm that specified requirements have been fulfilled as documented.
- **Compliance verification:** Ensure the test object complies with contractual, legal, and regulatory requirements.
- **Providing information:** Give stakeholders data to allow them to make informed decisions about releases.
- **Building confidence:** Establish trust in the quality of the test object before deployment.
- **Validating completeness:** Confirm the test object is complete and works as expected by stakeholders.
