**Learning Objectives:**

---

## 🔎 Overview

> Simply acquiring a tool does not guarantee success. Each new tool requires effort to achieve real and lasting benefits (e.g., for tool introduction, maintenance, and training).

---

## ✅ 6.2.1 Benefits of Test Automation `[K1]`

*FL-6.2.1: Recall the benefits and risks of test automation*

### Potential Benefits

| Benefit | Description |
|---------|-------------|
| **Time Saved** | Reducing repetitive manual work (e.g., execute regression tests, re-enter same test data, compare expected vs. actual results, check against coding standards) |
| **Greater Consistency and Repeatability** | Prevention of simple human errors (e.g., tests consistently derived from requirements, test data created systematically, tests executed in same order with same frequency) |
| **More Objective Assessment** | Providing measures that are too complicated for humans to determine (e.g., coverage) |
| **Easier Access to Information** | Supporting test management and test reporting (e.g., statistics, graphs, aggregated data about test progress, failure rates, test execution duration) |
| **Reduced Test Execution Times** | Providing earlier defect detection, faster feedback, and faster time to market |
| **More Time for Testers** | Freeing testers to design new, deeper, and more effective tests |

---

## ⚠️ Risks of Test Automation `[K1]`

### Potential Risks

| Risk | Description |
|------|-------------|
| **Unrealistic Expectations** | Unrealistic expectations about the benefits of a tool (including functionality and ease of use) |
| **Inaccurate Estimations** | Underestimating time, costs, effort required to introduce a tool, maintain test scripts, and change existing manual test process |
| **Inappropriate Use** | Using a test tool when manual testing is more appropriate |
| **Over-reliance on Tool** | Relying on a tool too much, e.g., ignoring the need for human critical thinking |
| **Vendor Dependency** | Dependency on tool vendor who may go out of business, retire the tool, sell to different vendor, or provide poor support (responses to queries, upgrades, defect fixes) |
| **Open-source Risks** | Using open-source software that may be abandoned (no further updates) or require frequent internal component updates |
| **Compatibility Issues** | Automation tool is not compatible with the development platform |
| **Unsuitable Tool Selection** | Choosing an unsuitable tool that does not comply with regulatory requirements and/or safety standards |

---

## 🎯 Exam Prep Tips

- **Benefits `[K1]`:** Recall the six benefits (time saved, consistency/repeatability, objective assessment, easier access to information, reduced execution times, more time for testers).
- **Risks `[K1]`:** Recall the eight risks (unrealistic expectations, inaccurate estimations, inappropriate use, over-reliance, vendor dependency, open-source risks, compatibility issues, unsuitable tool selection).
- **No Guarantee `[K1]`:** Remember that simply acquiring a tool does not guarantee success.
- **Effort Required `[K1]`:** Tools require effort for introduction, maintenance, and training.
- **Human Critical Thinking `[K1]`:** Automation should not replace the need for human critical thinking.
